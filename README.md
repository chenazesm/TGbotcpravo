# TGbotcpravo
## Состав файлов
main.py — серверная логика: управление сессиями пользователей, обработка событий Telegram Bot API и обслуживание запросов Mini App (Flask).

scenarios.json — структурированная база данных игровых уровней и сценариев угроз.
## Техническая спецификация
### Логика обработки контента (scenarios.json)
Поле "id" внутри объектов сценария является метаданным для уникальной идентификации записи. Фактическая последовательность прохождения игровых уровней определяется исключительно порядком расположения объектов в массиве (индексом массива), а не значением ключа "id".
### Среда выполнения и конфигурация (main.py)
Контейнеризация: Скрипт предназначен для запуска в подготовленной среде Docker. Прямой запуск без настройки сетевого окружения и зависимостей не гарантирует корректную работу.

Конфигурация: Все параметры (токены доступа, адреса БД, пути к файлам) считываются из переменных окружения (.env).

Зависимости: Требуется наличие активного соединения с СУБД Redis для кэширования и хранения состояний (state-management) игроков.

Сетевой стек: Для работы WebApp и Webhook требуется наличие SSL-сертификатов и Reverse Proxy (Nginx).
